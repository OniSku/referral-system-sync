# Referral System Sync

Распределенная система синхронизации реферальных начислений для Telegram ботов с централизованной обработкой бонусов.

## Описание

Реализует микросервисную архитектуру для отслеживания реферальных ссылок и распределения бонусов между несколькими VPN-сервисами. Независимые бот-сервисы взаимодействуют с центральным узлом обработки рефералов, ответственным за начисление бонусов.

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| Bot Framework | aiogram (Telegram Bot API) |
| База данных | SQLite |
| Контейнеризация | Docker & Docker Compose |
| Язык программирования | Python 3.9+ |
| Архитектура | Микросервисы |

## Архитектура

```
referral-system-sync/
├── bot_services/
│   ├── main_bot/          # Основной бот VPN-сервиса
│   │   └── handlers.py    # Обработчики команд бота
│   └── protocol_bot/      # Вторичный бот VPN-сервиса
│       └── handlers.py    # Обработчики команд бота
├── ref_center/
│   ├── main.py           # Центральный сервис координации
│   └── scanner/
│       └── sync.py       # Сканирование платежей и реферальная логика
└── docker-compose.yml    # Оркестрация сервисов
```

## Ключевые возможности

- **Универсальный перехватчик рефералов**: Захват реферальных ID сразу при запуске бота, предотвращающий потерю данных в процессе регистрации пользователей
- **Сканер платежей**: Автоматическое обнаружение и обработка транзакций с правильной логикой SQL JOIN
- **Кросс-сервисная синхронизация**: Централизованное отслеживание рефералов между несколькими независимыми сервисами
- **Целостность данных**: Исправленные SQL-запросы с использованием глобальных Telegram ID вместо внутренних ID таблиц
- **Поддержка форматов дат**: Совместимость с современными форматами баз данных (ISO 8601)
- **Обработка в реальном времени**: Мгновенное начисление бонусов после подтверждения платежа

## Установка и настройка

### Предварительные требования

- Docker и Docker Compose
- Python 3.9+
- SQLite3

### Переменные окружения

| Переменная | Описание | Обязательная |
|------------|----------|--------------|
| `BOT_TOKEN_MAIN` | Токен Telegram бота для основного сервиса | Да |
| `BOT_TOKEN_PROTOCOL` | Токен Telegram бота для протокольного сервиса | Да |
| `DB_PATH` | Путь к центральной базе данных | Да |
| `SCAN_INTERVAL` | Интервал сканирования платежей в секундах | Нет (по умолчанию: 300) |

### Быстрый старт

1. Клонируйте репозиторий:
```bash
git clone https://github.com/OniSku/referral-system-sync.git
cd referral-system-sync
```

2. Настройте переменные окружения в файле `.env`

3. Соберите и запустите сервисы:
```bash
docker-compose up --build
```

4. Сервисы начинают сканирование платежей и обработку рефералов при запуске

Данный проект решает критические проблемы интеграции в распределенных реферальных системах:
